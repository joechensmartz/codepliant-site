# v4.0 Spec — Multi-Jurisdiction Privacy Support

**Version:** 4.0
**Date:** 2026-03-15
**Status:** SPEC COMPLETE — Ready for Builder

---

## Overview

v4.0 adds multi-jurisdiction privacy compliance. The current privacy policy generator is GDPR-focused (EU only). This version extends it to support CCPA/CPRA (California), UK GDPR, and the 20 US state privacy laws now in effect. It also introduces a new document type (COMPLIANCE_NOTES.md) and a universal opt-out mechanism recommendation.

**Goal:** A user sets `jurisdictions: ["gdpr", "ccpa", "uk-gdpr"]` in config (or sets nothing and gets all applicable frameworks auto-detected), and the generated documents include jurisdiction-specific sections, rights, and language.

---

## 1. Jurisdiction System Design

### 1.1 Supported Jurisdiction Identifiers

| ID | Framework | Scope |
|----|-----------|-------|
| `gdpr` | EU General Data Protection Regulation | EU/EEA residents |
| `uk-gdpr` | UK General Data Protection Regulation | UK residents |
| `ccpa` | California Consumer Privacy Act / CPRA | California residents |
| `us-state` | Common US state privacy law provisions | 20 US states (see research-log.md) |

Future versions may add individual US state IDs (e.g., `vcdpa`, `cpa-co`), but v4.0 treats the 19 non-California states as a unified `us-state` bucket since their requirements are largely convergent.

### 1.2 Config Field

Add to `CodepliantConfig` and `GeneratorContext`:

```typescript
jurisdictions?: string[];    // e.g. ["gdpr", "ccpa", "uk-gdpr"]
companyLocation?: string;    // e.g. "US", "UK", "EU", "DE" — ISO country code
targetMarkets?: string[];    // e.g. ["US", "EU", "UK"] — where users are
```

### 1.3 Auto-Detection Logic

If `jurisdictions` is not set (undefined or empty array), auto-detect based on other signals:

1. If `companyLocation` is in EEA or `targetMarkets` includes any EU country -> add `gdpr`
2. If `companyLocation` is "UK" or `targetMarkets` includes "UK" -> add `uk-gdpr`
3. If `companyLocation` is "US" or `targetMarkets` includes "US" -> add `ccpa` + `us-state`
4. If none of the above are set -> **default to all jurisdictions** (`["gdpr", "uk-gdpr", "ccpa", "us-state"]`)

The fallback-to-all approach is intentional: most SaaS apps serve global users, and generating comprehensive coverage is safer than under-generating. Users can narrow it down via config.

### 1.4 Architecture: Additive Sections

Jurisdictions do NOT produce separate documents. Instead, each jurisdiction **adds or modifies sections** within the existing Privacy Policy, Cookie Policy, and Terms of Service. The pattern:

```
Base document (always generated)
  + GDPR sections (if "gdpr" in jurisdictions)
  + UK GDPR sections (if "uk-gdpr" in jurisdictions)
  + CCPA sections (if "ccpa" in jurisdictions)
  + US State sections (if "us-state" in jurisdictions)
```

This keeps output to a single Privacy Policy file, matching how real companies structure their policies (Stripe, Vercel, Linear all use a single document with jurisdiction-specific addenda at the end).

### 1.5 Jurisdiction Helper Module

Create `src/generator/jurisdictions.ts`:

```typescript
export type JurisdictionId = "gdpr" | "uk-gdpr" | "ccpa" | "us-state";

export interface ResolvedJurisdictions {
  ids: JurisdictionId[];
  hasGDPR: boolean;
  hasUKGDPR: boolean;
  hasCCPA: boolean;
  hasUSState: boolean;
}

export function resolveJurisdictions(
  jurisdictions?: string[],
  companyLocation?: string,
  targetMarkets?: string[]
): ResolvedJurisdictions;
```

This function encapsulates all auto-detection logic and returns a typed object that generators can check with simple boolean flags.

## 2. CCPA/CPRA Additions to Privacy Policy

When `ccpa` is in resolved jurisdictions, append these sections to the Privacy Policy.

### 2.1 Section: "Your California Privacy Rights (CCPA/CPRA)"

Top-level section added after the existing "Your Rights" section. Contains:

#### 2.1.1 Categories of Personal Information

Map detected service categories to CCPA categories. Use this mapping:

| Detected Category | CCPA Category (Cal. Civ. Code 1798.140(v)) |
|-------------------|---------------------------------------------|
| auth | Identifiers (name, email, account ID) |
| payment | Commercial information (transaction records, purchasing history) |
| analytics | Internet or electronic network activity (browsing history, search history, interaction data) |
| ai | Inferences drawn from other PI to create a profile |
| email | Identifiers (email address) |
| monitoring | Internet or electronic network activity (device info, error logs) |
| storage | Audio, electronic, visual, or similar information |
| advertising | Internet or electronic network activity; Inferences |
| social | Identifiers; Internet or electronic network activity |

Template text:

```markdown
### Categories of Personal Information Collected

Under the CCPA, we collect the following categories of personal information:

| CCPA Category | Examples | Source | Business Purpose |
|---------------|----------|--------|-----------------|
[auto-generated from detected services]

We have collected the above categories of personal information in the preceding 12 months.
```

#### 2.1.2 "Do Not Sell or Share My Personal Information"

```markdown
### Do Not Sell or Share My Personal Information

We do not sell your personal information as defined under the CCPA/CPRA.

We [do / do not] share your personal information for cross-context behavioral advertising purposes.

[If advertising services detected:]
We share the following categories of personal information for targeted advertising purposes:
- Internet or electronic network activity information (with [detected advertising service names])

You have the right to opt out of the sharing of your personal information for cross-context behavioral advertising. To exercise this right:
- [If GPC detected: "We honor the Global Privacy Control (GPC) signal. Enable GPC in your browser to automatically opt out."]
- Contact us at [email]
- [If applicable: Visit our "Do Not Sell or Share" page at [URL]]
```

**Logic:** If no advertising services are detected, use the "We do not sell or share" language. If advertising services ARE detected (category === "advertising"), switch to disclosure mode and list them.

#### 2.1.3 California Consumer Rights

```markdown
### Your Rights as a California Consumer

Under the CCPA/CPRA, California residents have the following rights:

1. **Right to Know:** You may request that we disclose the categories and specific pieces of personal information we have collected about you, the categories of sources, the business purposes for collection, and the categories of third parties with whom we share your information.

2. **Right to Delete:** You may request deletion of your personal information, subject to certain exceptions (e.g., completing a transaction, detecting security incidents, complying with legal obligations).

3. **Right to Correct:** You may request correction of inaccurate personal information.

4. **Right to Opt-Out of Sale/Sharing:** You may opt out of the sale or sharing of your personal information for cross-context behavioral advertising.

5. **Right to Limit Use of Sensitive Personal Information:** If we collect sensitive personal information, you may limit our use to purposes necessary to provide the service.

6. **Non-Discrimination:** We will not discriminate against you for exercising any of your CCPA/CPRA rights.

To exercise any of these rights, contact us at [email]. We will respond to verifiable consumer requests within 45 days.
```

#### 2.1.4 Right to Limit Use of Sensitive Personal Information

Only include if sensitive data categories are detected (auth with biometrics, geolocation, racial/ethnic origin, etc.). For v4.0, include this as a conditional paragraph within 2.1.3 if auth services are detected (since auth commonly processes email + password, which qualifies as account credentials under CCPA sensitive PI).

#### 2.1.5 Financial Incentive Disclosure

```markdown
### Financial Incentives

[If no loyalty/discount programs detected:]
We do not offer financial incentives for the collection, sale, or deletion of your personal information.

[If detected:]
We offer the following financial incentive programs that involve the collection of personal information:
- [Program description]
You can opt in to these programs by [method]. You can opt out at any time by [method]. The value of your personal information is reasonably related to the value of the offer provided.
```

For v4.0, default to "We do not offer financial incentives" since Codepliant does not currently detect loyalty/discount programs. Add a TODO comment for v5.0 smart scanning to detect this.

### 2.2 Modify Existing Sections

When CCPA is active, also modify:

- **"Your Rights" section heading:** Change to "Your Rights (General)" and add a note: "California residents: see the California Privacy Rights section below for additional rights under the CCPA/CPRA."
- **Third-party services section:** Add a sentence: "For details on the categories of personal information shared with each service provider, see the California Privacy Rights section."

## 3. UK GDPR Differences

When `uk-gdpr` is in resolved jurisdictions, add a "For UK Residents" section and modify certain existing sections.

### 3.1 Supervisory Authority

Replace/supplement EU supervisory authority references:

```markdown
### For UK Residents

If you are located in the United Kingdom, the following applies:

**Supervisory Authority:** You have the right to lodge a complaint with the Information Commissioner's Office (ICO):
- Website: https://ico.org.uk/make-a-complaint/
- Phone: 0303 123 1113
```

### 3.2 UK-Specific Lawful Basis Language

The legal basis table (currently titled "Legal Basis for Processing (GDPR Art. 6)") should be adjusted:

- If ONLY `uk-gdpr` (no `gdpr`): Title becomes "Legal Basis for Processing (UK GDPR Art. 6)"
- If BOTH `gdpr` and `uk-gdpr`: Title becomes "Legal Basis for Processing (GDPR / UK GDPR Art. 6)" — the articles are identical, so the table content stays the same.

### 3.3 UK International Transfer Mechanisms

When `uk-gdpr` is active, the international data transfers section must include UK-specific mechanisms:

```markdown
#### UK International Transfer Mechanisms

For transfers of personal data outside the United Kingdom, we rely on:

- **UK adequacy regulations** — transfers to countries the UK Secretary of State has determined provide adequate data protection
- **International Data Transfer Agreement (IDTA)** — the UK's equivalent of EU Standard Contractual Clauses, approved by the ICO
- **UK Addendum to the EU SCCs** — where we use EU SCCs, we include the UK Addendum issued by the ICO under s.119A of the Data Protection Act 2018
- **UK Extension to the EU-US Data Privacy Framework** — for transfers to certified US organisations
```

**Logic:** If both `gdpr` and `uk-gdpr` are active, show both EU and UK transfer mechanisms in the same section, clearly labeled.

### 3.4 Data Controller Reference

If `uk-gdpr` is active without `gdpr`, replace "European Economic Area (EEA)" references with "United Kingdom" in the international transfers section.

## 4. Universal Opt-Out Mechanism (Global Privacy Control)

### 4.1 Detection Logic

In `src/generator/jurisdictions.ts`, add:

```typescript
export function shouldRecommendGPC(jurisdictions: ResolvedJurisdictions): boolean {
  // GPC required by 11+ US states: CA, CO, CT, DE, MD, MN, MT, NH, NJ, OR, TX
  return jurisdictions.hasCCPA || jurisdictions.hasUSState;
}
```

### 4.2 Privacy Policy Addition

When GPC is recommended, add to the CCPA section (2.1.2 above) and to the general "Your Rights" section:

```markdown
### Universal Opt-Out Signals

We recognize and honor the Global Privacy Control (GPC) signal. When we detect a GPC signal from your browser, we treat it as a valid opt-out request for:
- Sale of personal information (where applicable)
- Sharing of personal information for cross-context behavioral advertising
- Targeted advertising

To enable GPC, visit https://globalprivacycontrol.org/ for browser extensions and supported browsers.

As of 2026, the following US states require recognition of universal opt-out mechanisms: California, Colorado, Connecticut, Delaware, Maryland, Minnesota, Montana, New Hampshire, New Jersey, Oregon, and Texas.
```

### 4.3 Cookie Policy Addition

When GPC is recommended, add to the Cookie Policy (modify `cookie-policy.ts`):

```markdown
### Global Privacy Control (GPC)

This site respects the Global Privacy Control (GPC) signal. If your browser sends a GPC signal, we will:
- Treat it as an opt-out of non-essential cookies used for advertising or cross-context behavioral tracking
- Not set advertising or tracking cookies until you affirmatively consent

We recommend enabling GPC in your browser for automatic privacy preference signaling across all websites that support it.
```

## 5. Jurisdiction-Specific Compliance Notes Document

### 5.1 New Document Type: COMPLIANCE_NOTES.md

Create `src/generator/compliance-notes.ts`.

This is a NEW document type — a practical checklist/reference that tells the developer which jurisdictions apply and what they need to do. It is NOT a legal document shown to users; it is an internal reference for the development team.

### 5.2 Document Structure

```markdown
# Compliance Notes

**Generated:** [date]
**Project:** [project name]

---

## Applicable Jurisdictions

| Jurisdiction | Framework | Why It Applies |
|-------------|-----------|----------------|
| EU/EEA | GDPR | [auto-detected reason or "Configured in .codepliantrc.json"] |
| UK | UK GDPR | ... |
| California | CCPA/CPRA | ... |
| US (20 states) | Various | ... |

---

## Action Items by Jurisdiction

### GDPR (EU/EEA)

- [ ] Appoint a Data Protection Officer (DPO) if required under Art. 37
- [ ] Establish a lawful basis for each processing activity
- [ ] Implement cookie consent mechanism compliant with ePrivacy Directive
- [ ] Ensure data subject access request (DSAR) process is in place (response within 30 days)
- [ ] Review international data transfer mechanisms (SCCs, adequacy decisions, DPF)
- [ ] Conduct Data Protection Impact Assessment (DPIA) for high-risk processing
- [ ] Maintain Records of Processing Activities (Art. 30)
[If AI services detected:]
- [ ] Prepare for EU AI Act compliance by August 2, 2026 (see AI_ACT_CHECKLIST.md)

### UK GDPR

- [ ] Register with the Information Commissioner's Office (ICO) if required
- [ ] Ensure UK Addendum or IDTA is in place for international transfers
- [ ] Review UK adequacy regulations for transfer destinations
- [ ] Verify lawful basis under UK GDPR Art. 6 (mirrors EU GDPR)
- [ ] Ensure DSAR response process meets 30-day requirement

### CCPA/CPRA (California)

- [ ] Provide "Do Not Sell or Share My Personal Information" link on homepage (if applicable)
- [ ] Implement verifiable consumer request process (respond within 45 days)
- [ ] Honor Global Privacy Control (GPC) signal
- [ ] Update Privacy Policy with CCPA-required disclosures (categories collected, sold, shared)
- [ ] Train customer-facing staff on CCPA consumer rights
- [ ] Review and update data inventory for CCPA categories
[If advertising services detected:]
- [ ] Implement opt-out mechanism for cross-context behavioral advertising
- [ ] Disclose categories of PI shared for advertising in Privacy Policy

### US State Privacy Laws (General)

- [ ] Honor universal opt-out mechanisms (required by 11+ states)
- [ ] Implement consumer rights process: access, correct, delete, opt-out
- [ ] Obtain opt-in consent before processing sensitive data
- [ ] Conduct data protection assessments for high-risk processing
- [ ] Review applicability thresholds for each state (see table below)

## State Applicability Thresholds

| State | Consumer Threshold | Revenue/Data Threshold |
|-------|-------------------|----------------------|
| California | 50,000+ households | $25M+ revenue OR 50%+ revenue from selling PI |
| Virginia | 100,000 consumers | OR 25,000 consumers + 50% revenue from PI |
| Colorado | 100,000 consumers | OR 25,000 consumers + revenue from PI |
| Rhode Island | 35,000 consumers | OR 10,000 consumers + 20% revenue from data sales |
| [other states] | 100,000 consumers | OR 25,000 consumers + revenue from PI (typical) |

> **Note:** Most states use similar thresholds. Rhode Island has the lowest. If you serve US users at any scale, assume at least some state laws apply.

---

## Detected Services and Jurisdiction Impact

[Table mapping each detected service to jurisdiction-specific implications]

| Service | Category | GDPR Impact | CCPA Impact | UK GDPR Impact |
|---------|----------|-------------|-------------|----------------|
[auto-generated from scan results]

---

*These compliance notes were generated by Codepliant based on automated code analysis. They are intended as a starting point — consult legal counsel for definitive compliance guidance.*
```

### 5.3 Generation Logic

- Always generate when any jurisdiction is resolved (i.e., almost always)
- Register in `src/generator/index.ts` with filename `COMPLIANCE_NOTES.md`
- The "Why It Applies" column uses:
  - "Configured in .codepliantrc.json" if explicitly set
  - "Company located in [location]" if auto-detected from `companyLocation`
  - "Target market includes [market]" if auto-detected from `targetMarkets`
  - "Default: generating for all major frameworks" if no config provided
- The "Detected Services" table cross-references scan results with jurisdiction requirements

## 8. Testing Requirements

### Unit Tests

1. **Jurisdiction resolution:**
   - Explicit `jurisdictions: ["gdpr"]` -> only GDPR
   - Empty array -> all jurisdictions (auto-detect fallback)
   - `companyLocation: "UK"` with no jurisdictions -> resolves to `uk-gdpr`
   - `targetMarkets: ["US"]` -> resolves to `ccpa` + `us-state`
   - No config at all -> all four jurisdictions

2. **CCPA sections in privacy policy:**
   - Advertising services detected -> "Do Not Sell" section mentions sharing
   - No advertising -> "We do not sell or share" language
   - California rights list includes all 6 rights
   - CCPA category mapping covers all detected service categories

3. **UK GDPR sections in privacy policy:**
   - ICO complaint reference present
   - UK transfer mechanisms listed (IDTA, UK Addendum)
   - Legal basis heading adjusts for UK-only vs UK+EU

4. **GPC recommendation:**
   - Present when CCPA or US-state jurisdiction active
   - Absent when only GDPR/UK-GDPR active
   - Appears in both Privacy Policy and Cookie Policy

5. **Compliance notes:**
   - Generated for any jurisdiction configuration
   - Contains correct "Why It Applies" reasoning
   - Action items present per active jurisdiction
   - Detected services table matches scan results

### Integration Tests

- Generate full document set with `jurisdictions: ["gdpr", "ccpa", "uk-gdpr", "us-state"]` — verify all sections present
- Generate with `jurisdictions: ["gdpr"]` only — verify no CCPA/UK sections
- Generate with no config — verify all jurisdictions included by default
- Compare output word count to target range (should not exceed 8,000 words for privacy policy with all jurisdictions)
