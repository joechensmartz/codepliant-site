## 4. Defects Found

### P0 — Must Fix

**DEF-1: Missing AI_ACT_CHECKLIST.md generation**
- No compliance checklist document is generated
- The EU AI Act requires organizations to demonstrate compliance steps taken
- Should generate a separate `AI_ACT_CHECKLIST.md` with actionable items (risk assessment, FRIA, conformity assessment, documentation requirements, etc.)

**DEF-2: companyName not used in AI Disclosure**
- `src/generator/ai-disclosure.ts` reads `ctx?.contactEmail` but never reads `ctx?.companyName`
- The generated document has no company/organization name anywhere
- Fix: Add company name to the header and Contact section

### P1 — Should Fix

**DEF-3: Art. 50(2) synthetic content marking not explicit**
- Section 4.1 alludes to AI-generated content but does not:
  - Explicitly cite Art. 50(2) of the EU AI Act
  - Require machine-readable marking of synthetic content
  - Mention labeling obligations for deployers
- Should add a subsection like "4.4 Synthetic Content Marking (Art. 50(2))" covering:
  - Obligation to mark AI-generated text/image/audio/video
  - Technical implementation guidance
  - Exception for editorially reviewed content

**DEF-4: No DPO contact in AI Disclosure**
- Config has `dpoEmail: "dpo@aicorp.eu"` but AI Disclosure only shows contactEmail
- For GDPR-covered entities, the DPO should also be listed as a contact for AI-related queries

### P2 — Nice to Have

**DEF-5: Generic "Purpose of AI Usage" section**
- Section 2 lists generic purposes and tells the user to customize
- Could be improved by inferring purpose from detected services (e.g., OpenAI -> "Natural language processing and content generation")

**DEF-6: No Art. 52 chatbot disclosure template**
- If the app uses conversational AI (OpenAI, Anthropic), Art. 52 requires disclosure that users are interacting with an AI
- Could generate a code snippet or UI text template for in-app disclosure

**DEF-7: Contact section minimal**
- Only shows email. Could include company name, DPO email, physical address placeholder, and supervisory authority reference

---

## 5. Summary

The existing AI Disclosure generator is a good foundation — it correctly detects AI services, references the EU AI Act with the regulation number, classifies risks, lists provider policies, and covers user rights. However, for a v3.0 "EU AI Act Compliance" release, it needs:

1. **AI_ACT_CHECKLIST.md** — A separate actionable compliance checklist document
2. **companyName integration** — Use the configured company name in the document
3. **Art. 50(2) explicit coverage** — Synthetic content marking obligations
4. **DPO contact** — Include DPO email when configured
5. **Richer contact section** — Company name, DPO, address placeholder

The edge case (no AI = no disclosure) works correctly. All 100 existing tests pass with no regressions.
