## Analysis

### Stability
- **Zero crashes across all 200 repos** -- scanner remains rock solid
- Handled repos of all sizes: from tiny libs (cobra, serde) to massive monorepos (kubernetes, tensorflow, rust-lang/rust, cockroachdb)
- 3 clone failures were GitHub-side (repo moved/renamed), not scanner issues

### Language Coverage Insights
- **Ruby apps score highest** (4.33 avg) -- e-commerce apps (spree, solidus) and full-stack apps (discourse) have many integrations the scanner picks up well
- **Elixir apps surprisingly strong** (4.11 avg) -- plausible (14), papercups (12), realtime (7) detected well, likely via their JS/TS frontends and config files
- **Go apps solid** (2.17 avg) -- good detection on apps with config files (vault, cockroach, grafana, rclone)
- **Pure Go/Rust/Java libraries score 0** as expected -- no third-party services in stdlib-style packages
- **PHP utility libs underdetected** -- WordPress (0), Drupal (0) likely need deeper PHP scanning

### Zero-Detection Repos Worth Investigating
These repos almost certainly use third-party services but returned 0:
- **kubernetes/kubernetes** -- uses etcd, cloud APIs (AWS/GCP/Azure), metrics systems
- **apache/airflow** -- workflow tool with 1000+ provider integrations
- **elastic/elasticsearch** -- uses S3, cloud storage, LDAP
- **WordPress/WordPress** -- massive plugin ecosystem, uses MySQL, SMTP
- **Rocket.Chat** -- uses MongoDB, push notifications, OAuth providers
- **ansible/ansible** -- uses SSH, cloud providers, LDAP

### Recommendations
1. **Improve Go scanning** -- detect service usage via Go import paths and go.mod
2. **Add Java/Maven scanning** -- parse pom.xml for known service dependencies
3. **Improve PHP scanning** -- parse composer.json more thoroughly
4. **Add Rust/Cargo.toml scanning** -- detect crates like `rusoto_*`, `aws-sdk-*`
5. **Add Elixir/mix.exs scanning** -- detect hex packages for services
