## BLOCKING BUGS (will embarrass in demo)

### Bug #1: Version mismatch -- shows v12.0.0, should be v15.0.0
- **File:** `src/cli.ts:26` -- `const VERSION = "12.0.0"`
- **Also:** `package.json` says `"version": "12.0.0"`
- **Impact:** Investor sees wrong version number in every command's banner
- **Fix:** Update both to `"15.0.0"`

### Bug #2: `check` command false FAIL -- PCI DSS Compliance
- **Symptom:** `check` exits 1 with "Missing required document(s): PCI DSS Compliance"
- **Root cause:** The `docFileMap` in `runCheck()` (cli.ts:1028-1034) only maps 5 document types but `complianceNeeds` can return "PCI DSS Compliance" and "Accessibility (WCAG 2.1 / ADA)". These have no filename mapping, so they always show as MISSING.
- **Impact:** Demo of `check` command always fails when Stripe is detected. Investor sees red FAIL.
- **Fix:** Add PCI and Accessibility entries to `docFileMap`, or only fail on documents that have known filenames.

### Bug #3: `diff` command shows false removals
- **Symptom:** After `go --format all`, running `diff` (without `--format all`) shows `compliance.html`, `index.html`, etc. as "would be removed (service removed from codebase)"
- **Root cause:** `runDiff()` (cli.ts:1109) calls `generateDocuments()` which only generates markdown by default. The HTML files on disk don't appear in the new doc set, so they're flagged as removed.
- **Impact:** Investor sees scary red "would be removed" lines for files that were just generated.
- **Fix:** Pass the format flag through to diff's internal generation, or exclude non-markdown files from removal detection.

### Bug #4: Circular self-detection in generated HTML
- **Symptom:** First scan detects 12 services. After `go --format all`, subsequent scans detect 13 services because the generated `index.html`/`compliance.html` contain Google Analytics tracking code (`gtag()`) that the scanner picks up.
- **Root cause:** `legal/` directory is not in `IGNORE_DIRS` in `file-walker.ts:4-21`. Scanner walks into its own output.
- **Impact:** Service count inflates on re-scan. Compliance score drops from 85% to 73% between runs. Investor would notice changing numbers.
- **Fix:** Add `"legal"` to `IGNORE_DIRS` in `src/scanner/file-walker.ts`.

## COSMETIC ISSUES

### Issue #5: Banner box misaligned
- The `╔═══╗` top border is 39 chars wide, but the content lines (`║ ... ║`) are 41-43 chars wide.
- The right `║` doesn't align with `╗`/`╝`. Visible in every command.
- **File:** `src/cli.ts:34-39`

### Issue #6: Phantom Sentry services (4 false positives)
- Only `@sentry/node` is in `package.json`, but the scanner also reports `@sentry/nextjs`, `@sentry/react`, `@sentry/nestjs`, `@sentry/profiling-node` because they all share the `SENTRY_DSN` env var pattern.
- Similarly, `aws-sdk` triggers `@aws-sdk/client-s3`, `@aws-sdk/client-ses`, `@aws-sdk/client-sns`.
- An investor might ask: "Why does it say I'm using Sentry for React when I'm not?"

### Issue #7: PCI DSS priority inconsistency
- First run: PCI DSS shows as "RECOMMENDED" (85% score)
- Second run (after self-detecting GA in generated HTML): PCI DSS shows as "REQUIRED" (73% score), likely because the generated HTML contains credit-card-related text that triggers `PCI_FIELD_PATTERNS`.
- Non-deterministic behavior across runs.
