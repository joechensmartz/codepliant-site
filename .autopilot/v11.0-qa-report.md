## Bugs Found

### P1: JSON stdout truncation at 64KB
- **Impact:** `--json` output via pipe truncates at 65,536 bytes for large repos
- **Affected repos:** dub, formbricks, posthog
- **Workaround:** Redirect to file (`> output.json`)
- **Fix:** Use `process.stdout.write()` with proper chunking or stream the JSON

### P2: No Java/Maven detection
- **Impact:** Kill Bill (billing platform) detected 0 services
- **Root cause:** pom.xml dependency parsing not implemented or catalog missing Java libraries

### P3: No Go library catalog depth
- **Impact:** PocketBase detected 0 services despite having oauth2, mailyak
- **Root cause:** Go service catalog too narrow

### P3: False positive on reqwest (Rust)
- **Impact:** reqwest (HTTP client) flagged as a service, but it's a utility library
- **Root cause:** reqwest is in catalog but shouldn't be -- it's not a third-party data processor

### P4: Misleading compliance score for zero-detection repos
- **Impact:** PocketBase shows 100% compliance despite detecting nothing
- **Suggestion:** Show "insufficient data" or lower confidence when 0 services found

## Recommendations for v12.0

1. **Fix JSON stdout buffer** -- Stream JSON output or use larger buffer
2. **Expand Java catalog** -- Add Maven artifacts: stripe-java, braintree, redis.clients/jedis, aws-java-sdk, sentry-java
3. **Expand Go catalog** -- Add: mailyak, golang.org/x/oauth2, go-redis, aws-sdk-go-v2, google-cloud-go
4. **Remove reqwest from service catalog** -- It's an HTTP client, not a data processor
5. **Add PHP PayPal detection** -- paypal/paypal-checkout-sdk in composer.json
6. **Improve compliance score logic** -- Don't show 100% when 0 services detected
7. **Extract project metadata** -- Use package.json name/author/description to fill placeholders
