## Root Cause: Integration Bug

**File:** `/Users/joe/Desktop/idea/codepliant/src/scanner/index.ts`

The `scan()` function imports and calls:
- `scanDependencies` (package.json) -- YES
- `scanPythonDependencies` -- YES
- `scanGoDependencies` -- YES
- `scanImports` -- YES
- `scanEnvFiles` -- YES
- `scanPrismaSchema` -- YES

**Missing:**
- `scanRubyDependencies` from `./ruby.js` -- NOT IMPORTED, NOT CALLED
- `scanElixirDependencies` from `./elixir.js` -- NOT IMPORTED, NOT CALLED

The scanners compile, the unit tests pass (for their own files), but they are never invoked by the main pipeline.

## Prisma Schema Detection Bug

**Status:** Partially working but misses common monorepo layouts.

The `findPrismaSchemas()` function in `schema.ts` searches for `packages/*/prisma/schema.prisma`, which expects the schema at e.g. `packages/database/prisma/schema.prisma`. But both Cal.com and Documenso place theirs at `packages/prisma/schema.prisma` (the package IS the prisma package, schema is at root).

**Fix needed:** Also check `packages/*/schema.prisma` in addition to `packages/*/prisma/schema.prisma`.

- **File:** `src/scanner/index.ts`

- **Impact:** Would fix Maybe (6% -> ~75%) and Discourse (0% -> ~60%)

- Both `ruby.ts` and `elixir.ts` have no corresponding `.test.ts` files
