## Known Issues (not v4.0 regressions)

1. **`posthog-js` not detected** — Scanner only recognizes `posthog` package name. `posthog-js` (the browser SDK) is not in SERVICE_SIGNATURES. This means scenarios using `posthog-js` won't get Cookie Policy generated. (Pre-existing: BUG-8 scoped package matching)
2. **Auto-detect CCPA** — When no config is provided, CCPA is only auto-triggered by analytics/advertising services. Without those detected, CCPA won't appear even if the company is US-based. This is acceptable since there's no way to know company location without config.
