## 2. Competitor Website Analysis

### Termly (termly.io)

**Traffic:** Competitor data suggests Termly is in the 300K-500K monthly visits range. Competitors TermsFeed (~411K) and freeprivacypolicy.com (~533K) give a sense of the market size.

**Site Structure:**
- `/products/privacy-policy-generator/` -- core tool page, ranks #1 for many generator keywords
- `/products/terms-and-conditions-generator/`
- `/products/cookie-consent-manager/`
- `/resources/templates/` -- free templates hub (privacy policy template, ToS template, etc.)
- `/resources/articles/` -- educational blog with 100+ articles on compliance topics
- `/resources/compare/` -- comparison pages ("best privacy policy generators")
- `/pricing/` -- freemium to paid tiers

**What Works:**
- Template pages rank for "[document type] template" keywords
- Comparison/listicle pages they write rank for "best [tool] generator"
- Product pages target "generator" keywords directly
- Blog covers every privacy law (GDPR, CCPA, CalOPPA) with dedicated articles
- AI-disclosure generator launched to capture new keyword cluster

**Lesson for Codepliant:** Termly's dominance comes from (1) tool pages that ARE the product, (2) massive educational content moat, and (3) template pages that capture informational intent.

---

### Vanta (vanta.com)

**Positioning:** "Automate compliance. Simplify security." Targets startups needing SOC 2/ISO 27001.

**Site Structure:**
- `/products/automated-compliance` -- core product page
- `/products/ai` -- AI-specific compliance features
- `/landing/home-es`, `/landing-fr/compliance-automation` -- localized landing pages
- `/compare/drata` -- direct competitor comparison pages
- `/resources/how-does-vanta-work` -- educational content
- Trust center product page

**What Works:**
- Strong positioning around "90% automation" and "weeks instead of months"
- Specific numbers build credibility ("Used by 8,000+ companies")
- Comparison pages capture "Vanta vs Drata" searches
- Framework-specific landing pages (SOC 2, ISO 27001, HIPAA each get their own page)

**Lesson for Codepliant:** Create framework-specific pages. Every compliance framework should be a landing page.

---

### Drata (drata.com)

**Positioning:** Continuous compliance automation platform. Enterprise-focused.

**Site Structure:**
- Framework-specific pages for each of 20+ frameworks
- `/blog/` -- regular content covering regulation updates
- Integration pages for each tool (AWS, GitHub, Okta)
- Comparison pages vs competitors

**What Works:**
- Deep framework coverage (20+ frameworks = 20+ landing pages)
- Blog covers regulatory updates (e.g., "AI Regulations: State and Federal AI Laws 2026")
- Integration pages capture "[tool] + compliance" searches
- Real-time regulatory content keeps site fresh for Google

**Lesson for Codepliant:** Regulatory update content (new laws, deadline reminders) drives repeat traffic and freshness signals.

---

### Privacy Policy Generators (freeprivacypolicy.com, privacypolicies.com, termsfeed.com)

**Combined Traffic:** 1M+ monthly visits across the top 3 generators.

**What Works:**
- The tool IS the landing page -- users can start generating immediately
- SEO titles are extremely direct: "Free Privacy Policy Generator"
- One-time pricing (TermsFeed) vs subscription (Termly) creates differentiation
- Template download pages capture informational searches
- "Best privacy policy generator" listicle pages (written by the generators themselves) rank well

**Lesson for Codepliant:** Having a free, usable tool on the website is the single best SEO asset. The generator page itself becomes the top-ranking page.

### Open Source Precedents: shadcn/ui and Cal.com

**shadcn/ui Growth Strategy:**
- Released March 2023, reached 90K+ GitHub stars and 250K+ weekly npm installs by 2025
- NO traditional marketing. Growth was entirely through:
  1. Solving a real developer pain point (copy-paste components vs black-box packages)
  2. Exceptional README as landing page
  3. Ecosystem expansion (community-built templates, Figma kits, starter kits)
  4. Creator joined Vercel, gained distribution through v0
  5. Developer word-of-mouth on Twitter/X and dev communities

**Cal.com Growth Strategy:**
- Open-source scheduling ("Open Scheduling Infrastructure")
- B2C bottom-up: each user shares their link 50-100 times in 6 months; 10% convert to new users
- Handbook published publicly -- transparency as marketing
- Positioned for regulated industries (gov, healthcare) that NEED self-hosting
- Philosophy: "Create as much value as possible, capture a small amount from enterprise"
- Category creation: not "Calendly alternative" but "Open Scheduling"

**Lessons for Codepliant:**
1. Create a category: "Code-Aware Compliance" or "Compliance-as-Code" rather than "privacy policy generator alternative"
2. README = marketing page. Treat GitHub README with same care as homepage
3. Build viral loops: generated compliance documents can include "Generated by Codepliant" links
4. Target regulated industries that prefer open-source/self-hosted
5. Publish your methodology transparently (how scanning works, what rules apply)

## 3. Content Strategy Recommendation

### Phase 1: Foundation (Month 1-2)

**Core Pages (SEO-critical):**

3. **Terms of Service Generator** -- Same pattern
4. **AI Disclosure Generator** -- First-mover advantage. Target "AI disclosure template"
5. **How It Works** -- Technical explainer targeting "code compliance scanner," "automated compliance documentation"

**Framework Landing Pages:**
- `/compliance/gdpr` -- "GDPR Compliance for Developers"
- `/compliance/ccpa` -- "CCPA Compliance Tool"
- `/compliance/eu-ai-act` -- "EU AI Act Compliance" (high-growth keyword)
- `/compliance/california-ai-transparency-act` -- new law, low competition

- `/templates/ai-disclosure`

**Blog Content (target long-tail keywords):**

- "How to Generate a Privacy Policy from Your Codebase"
- "GDPR Requirements for Open Source Projects"
- "AI Disclosure Requirements: California, Colorado, and EU Laws Explained"
- "What Privacy Laws Apply to Your App? A Developer's Guide"
- "Privacy Policy vs Terms of Service: What's the Difference?"
- "Evidence-Based Compliance: Why Scanning Code Beats Writing Policies from Scratch"

**Comparison Pages:**
- "/compare/termly" -- "Codepliant vs Termly: Code-Aware vs Form-Based"
- "/compare/privacy-policy-generators" -- "Best Privacy Policy Generators for Developers 2026"

### Phase 3: Authority Building (Month 4+)

- Regulatory update blog posts (track new state privacy laws, EU AI Act deadlines)
- Developer guides and tutorials
- Integration guides (Next.js, Rails, Django, Express)
- Case studies from early adopters
- Guest posts on dev blogs (Dev.to, Hashnode, Smashing Magazine)

### Content Differentiation Angle

Codepliant's unique value is that it SCANS CODE to generate documents. Every piece of content should reinforce this:
- "Most privacy policy generators ask you to fill out forms. Codepliant reads your code."
- "Your code already knows what data you collect. Codepliant makes it official."
- This is the wedge that separates Codepliant from Termly/TermsFeed/etc.

## 4. Technical SEO Checklist for the Marketing Site

### Performance
- [ ] Page load under 2 seconds (pages loading in 1s convert 3x better than 5s)
- [ ] Core Web Vitals passing (LCP < 2.5s, FID < 100ms, CLS < 0.1)
- [ ] GZIP/Brotli compression enabled (reduces file sizes by up to 70%)
- [ ] Image optimization (WebP/AVIF, lazy loading)
- [ ] Minimal JavaScript bundle (static site preferred -- Next.js SSG or Astro)
- [ ] CDN for global delivery

### Crawlability & Indexing
- [ ] XML sitemap at `/sitemap.xml` submitted to Google Search Console
- [ ] `robots.txt` properly configured
- [ ] Canonical URLs on all pages
- [ ] No orphan pages (every page reachable within 3 clicks from homepage)
- [ ] Proper 301 redirects for any URL changes
- [ ] Internal linking strategy (tool pages link to related blog posts and vice versa)

### On-Page SEO
- [ ] Unique title tags per page (under 60 characters, keyword-first)
- [ ] Meta descriptions per page (under 155 characters, include CTA)
- [ ] H1 tags containing primary keyword on every page
- [ ] Structured data / JSON-LD (Organization, SoftwareApplication, FAQ, HowTo)
- [ ] Open Graph and Twitter Card meta tags for social sharing
- [ ] Alt text on all images

### Mobile
- [ ] Mobile-first design (63% of web traffic is mobile in 2026)
- [ ] Responsive layout, no horizontal scroll
- [ ] Tap targets sized appropriately (48px minimum)
- [ ] Mobile page speed under 3 seconds on 4G

### Content & Authority
- [ ] Blog with regular publishing cadence (2-4 posts/month minimum)
- [ ] Author pages with bios (E-E-A-T signals)
- [ ] Last-updated dates on all compliance content
- [ ] External links to authoritative sources (official regulation texts)
- [ ] Internal linking from blog posts to tool/product pages

### Technical Infrastructure
- [ ] HTTPS everywhere
- [ ] Clean URL structure (no query parameters for content pages)
- [ ] 404 page with navigation back to key pages
- [ ] Hreflang tags if multi-language content is added later
- [ ] Analytics (Plausible or PostHog for privacy-respecting analytics)
- [ ] Google Search Console verified

### Link Building
- [ ] GitHub README links to marketing site
- [ ] npm/package registry links to site
- [ ] Product Hunt launch
- [ ] Dev.to / Hashnode cross-posts
- [ ] "Powered by Codepliant" backlinks from generated documents

### Homepage Structure (above the fold)

Based on 2026 landing page best practices:

1. **Headline:** Clear, specific, keyword-rich (e.g., "Generate Compliance Documents from Your Codebase")
2. **Subheadline:** Differentiation (e.g., "Codepliant scans your code to create accurate privacy policies, terms of service, and AI disclosures. Open source.")
3. **Primary CTA:** `npm install -g codepliant` or "Try It Free" button
4. **Secondary CTA:** "View on GitHub" with star count
5. **Demo/Visual:** Terminal animation showing the CLI in action or before/after (code scan -> generated document)
6. **Social Proof:** GitHub stars, npm downloads, contributor count
7. **Framework logos:** GDPR, CCPA, EU AI Act, SOC 2 badges

### Key Design Principles

- **Mobile-first:** Design for phone first, expand to desktop
- **Speed over flash:** Static site generator (Astro, Next.js SSG), minimal JS
- **Tool pages are the product:** Users should be able to see/try the tool without leaving the page
- **Specificity over generality:** "Used by 847 developers" beats "Used by many developers"
- **One CTA per page section:** Don't dilute attention
