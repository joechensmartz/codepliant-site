**Duration:** 45 minutes | **Frequency:** Quarterly refresher

### 3.2 Password Hygiene & Authentication

**Duration:** 30 minutes | **Frequency:** Annually + onboarding

#### Learning Objectives

- [ ] Create strong, unique passwords (16+ characters, passphrase method)
- [ ] Use a company-approved password manager for all credentials
- [ ] Enable multi-factor authentication (MFA) on all accounts
- [ ] Never reuse passwords across services
- [ ] Recognise and avoid credential stuffing risks

#### Password Policy Summary

| Requirement | Standard |
|-------------|----------|
| Minimum length | 16 characters |
| Complexity | Mix of character types OR passphrase |
| Rotation | Not required if MFA enabled (NIST 800-63B) |
| MFA | Required for all corporate accounts |
| Password manager | Required — company-provided |
| Shared credentials | Prohibited — use SSO/service accounts |

### 3.3 Incident Reporting

**Duration:** 30 minutes | **Frequency:** Annually + onboarding

#### Learning Objectives

- [ ] Define what constitutes a security incident
- [ ] Know the internal reporting channels and timelines
- [ ] Understand that timely reporting is a duty, not optional
- [ ] Preserve evidence (do not delete emails, logs, screenshots)
- [ ] Know the escalation path for different incident types

#### Reporting Procedure

```
1. STOP — Do not attempt to fix the issue yourself
2. REPORT — Immediately contact:
   - Email: [your-email@example.com]
   - Slack: #security-incidents (if available)
   - Phone: [Security team phone number]
3. PRESERVE — Keep all evidence (emails, screenshots, URLs)
4. DOCUMENT — Note the time, what you observed, and what you did
5. WAIT — Do not discuss the incident outside the response team
```
