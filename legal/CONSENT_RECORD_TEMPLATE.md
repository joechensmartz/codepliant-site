## 6. GDPR Article 7 Compliance Checklist

Ensure each consent record meets these requirements:

- [ ] **Demonstrable (Art. 7(1))** — You can prove consent was given (timestamp, IP, method, exact text)
- [ ] **Freely given** — Consent is not bundled with T&Cs; separate consent for each purpose
- [ ] **Specific** — Each consent type maps to a specific, named processing purpose
- [ ] **Informed** — The consent text clearly identifies the controller and processing purposes
- [ ] **Unambiguous** — Consent requires a clear affirmative action (no pre-ticked boxes)
- [ ] **Withdrawable (Art. 7(3))** — Withdrawal is as easy as giving consent
- [ ] **Recorded** — All consent events (grant, modify, withdraw) are immutably logged
- [ ] **Versioned** — The policy version shown at consent time is recorded
- [ ] **Age-verified** — For children, parental consent is obtained per Art. 8

## 8. Implementation Guidance

### Database Schema (Example)

```sql
CREATE TABLE consent_records (
  consent_id      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         VARCHAR(255) NOT NULL,
  timestamp       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  consent_type    VARCHAR(50) NOT NULL,
  consent_action  VARCHAR(20) NOT NULL CHECK (consent_action IN ('granted', 'withdrawn', 'modified', 'expired')),
  ip_address      INET NOT NULL,
  user_agent      TEXT NOT NULL,
  method          VARCHAR(30) NOT NULL,
  policy_version  VARCHAR(20) NOT NULL,
  consent_text    TEXT NOT NULL,
  language        VARCHAR(5) NOT NULL DEFAULT 'en',
  expiry_date     TIMESTAMPTZ,
  parent_consent_id UUID REFERENCES consent_records(consent_id),
  source_url      TEXT,
  geo_location    VARCHAR(10),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_consent_user ON consent_records(user_id);
CREATE INDEX idx_consent_type ON consent_records(consent_type);
CREATE INDEX idx_consent_timestamp ON consent_records(timestamp);
```
