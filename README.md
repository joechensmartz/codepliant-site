## Quick Start

```bash
npx codepliant go
```

That's it. Codepliant scans your project, detects third-party services, and generates compliance documents in `./legal/`.

```bash
# Or install globally
npm install -g codepliant

# Or as a dev dependency
npm install --save-dev codepliant
```

## Features

### 10+ Ecosystems

Codepliant scans dependency manifests, source code imports, and environment variables across all major ecosystems:

| Ecosystem | Files Scanned |
|-----------|--------------|
| **JavaScript / TypeScript** | `package.json`, source imports, `.env` |
| **Python** | `requirements.txt`, `pyproject.toml`, `Pipfile`, source imports |
| **Go** | `go.mod`, `go.sum`, struct tags, source imports |
| **Ruby** | `Gemfile`, `Gemfile.lock`, source imports |
| **Elixir** | `mix.exs`, `mix.lock`, source imports |
| **PHP** | `composer.json`, `composer.lock`, source imports |
| **Rust** | `Cargo.toml`, `Cargo.lock`, source imports |
| **Java** | `pom.xml`, `build.gradle`, source imports |
| **.NET** | `.csproj`, `packages.config`, source imports |
| **Django** | `models.py`, `settings.py`, ORM analysis |

Plus cross-ecosystem scanners for environment variables, API routes, tracking pixels, Prisma schemas, SQLAlchemy models, Go structs, Django models, and data flow analysis.

### 12 Document Types

| Document | Filename | When Generated |
|----------|----------|----------------|
| **Privacy Policy** | `PRIVACY_POLICY.md` | Any user data collection detected |
| **Terms of Service** | `TERMS_OF_SERVICE.md` | Always generated |
| **Security Policy** | `SECURITY.md` | Always generated |
| **AI Disclosure** | `AI_DISCLOSURE.md` | AI services detected (EU AI Act ready) |
| **AI Act Compliance Checklist** | `AI_ACT_CHECKLIST.md` | AI services detected |
| **Cookie Policy** | `COOKIE_POLICY.md` | Analytics or auth cookies detected |
| **Data Processing Agreement** | `DATA_PROCESSING_AGREEMENT.md` | Third-party data processors detected |
| **Sub-Processor List** | `SUBPROCESSOR_LIST.md` | 3+ third-party services detected |
| **Data Flow Map** | `DATA_FLOW_MAP.md` | Any services detected |
| **Compliance Notes** | `COMPLIANCE_NOTES.md` | Regulation overview based on detected services |
| **Environment Variable Audit** | `ENV_AUDIT.md` | `.env` files found (secret detection, risk analysis) |
| **Custom Plugin Documents** | (varies) | Via plugin system |

### 7 Output Formats

| Format | Description |
|--------|-------------|
| `markdown` | Individual `.md` files per document (default) |
| `html` | Single self-contained `index.html` with tabs for each document |
| `pdf` | PDF file for each document |
| `json` | Structured JSON with scan results and document content |
| `all` | All of the above combined |
| `badge` | Shields.io-style SVG compliance badges |
| `widget` | Embeddable HTML widget for your website |

### 4 Languages

Generated documents are available in:

- English (`en`)
- Spanish (`es`)
- French (`fr`)
- German (`de`)

Set `"language": "de"` in your `.codepliantrc.json` to generate documents in German.

### Industry Compliance Detection

Codepliant automatically detects industry-specific compliance requirements:

- **HIPAA** ŌĆö detects healthcare packages (FHIR, HL7, pydicom), PHI field patterns, and health-related env vars
- **PCI DSS** ŌĆö flags payment processing services and card data handling
- **COPPA** ŌĆö detects age-gating patterns and child-directed content indicators

### Framework-Implicit Detection

Detects services implicitly bundled by frameworks (e.g., Next.js includes Vercel Analytics, Django includes session cookies) without explicit imports.

### Monorepo Support

Automatically detects and scans monorepo workspaces:
- npm/yarn workspaces
- pnpm workspaces
- Lerna
- Turborepo
- Directory conventions (`packages/`, `apps/`)

Evidence is tagged with workspace-relative paths so you know which package uses what.

### Plugin System

Extend Codepliant with custom scanners, generators, and service signatures:

```json
{
  "plugins": ["codepliant-plugin-healthcare"]
}
```

Plugins can:
- Register new service signatures for detection
- Add custom scanners (e.g., scan DICOM headers)
- Add custom document generators

See the [plugin API reference](#plugin-api) below.

### Environment Variable Audit

Scans `.env` and `.env.example` files to:
- Classify variables (secret, database, service-config, public)
- Flag hardcoded secrets and missing `.env.example` entries
- Generate a risk report with severity levels

### Accessibility Audit

Detects accessibility tooling (`axe-core`, `react-aria`, `@radix-ui/*`, `pa11y`), ARIA usage patterns, and linting rules (`eslint-plugin-jsx-a11y`). Generates compliance needs for WCAG/ADA when relevant.

### Data Flow Mapping

Traces how user data flows through your application: collection points (API routes, forms), storage (databases, file storage), processing (AI services, email), and third-party sharing. Generates a visual data flow document.

### 120+ Service Detections

Codepliant recognizes services across these categories:

- **AI** ŌĆö OpenAI, Anthropic, Google AI, Replicate, Together AI, Cohere, Pinecone, LangChain, Hugging Face, ChromaDB
- **Payment** ŌĆö Stripe, PayPal, Lemon Squeezy
- **Analytics** ŌĆö Google Analytics, PostHog, Mixpanel, Amplitude, Vercel Analytics, Hotjar
- **Auth** ŌĆö NextAuth, Clerk, Supabase Auth, Auth.js, better-auth, Passport, Django Allauth, Flask-Login
- **Email** ŌĆö SendGrid, Resend, Nodemailer, Postmark
- **Database** ŌĆö Prisma, Drizzle, Mongoose, SQLAlchemy, MongoEngine, psycopg2
- **Storage** ŌĆö AWS S3, UploadThing, Cloudinary, Google Cloud Storage, boto3
- **Monitoring** ŌĆö Sentry, Datadog
- **Advertising** ŌĆö Google Ads, Facebook Pixel, tracking pixels
- **Social** ŌĆö Social login providers, sharing integrations

## CLI Reference

### Scanning

```bash
# Scan + generate all documents
codepliant go [path]

# Scan only (no document generation)
codepliant scan [path]

# Quick pass/fail compliance check (for CI)
codepliant check [path]

# Show what changed since last generation
codepliant diff [path]
```

### Generation

```bash
# Generate compliance page (single HTML with tabs)
codepliant page [path]

# Generate compliance badge SVGs
codepliant badge [path]
```

### Setup

```bash
# Interactive setup wizard
codepliant init

# Install pre-commit hook
codepliant hook install

# Remove pre-commit hook
codepliant hook uninstall
```

### Options

| Flag | Short | Description |
|------|-------|-------------|
| `--output <dir>` | `-o` | Output directory (default: `./legal`) |
| `--format <fmt>` | | `markdown`, `html`, `pdf`, `json`, `all` |
| `--json` | | Output scan results as JSON |
| `--quiet` | `-q` | Minimal output |
| `--watch` | `-w` | Watch for file changes and regenerate |
| `--verbose` | `-v` | Per-scanner timing breakdown |
| `--version` | `-V` | Print version |
| `--help` | `-h` | Show help (works per-command too) |

### Examples

```bash
# Scan and generate HTML output
codepliant go --format html

# Generate all formats
codepliant go --format all

# Watch mode ŌĆö auto-regenerate on file changes
codepliant go --watch

# JSON scan results for scripting
codepliant scan --json | jq '.services'

# CI pipeline check
codepliant check --json

# Verbose timing breakdown
codepliant scan --verbose
```

## Configuration

Create a `.codepliantrc.json` in your project root (or run `codepliant init`):

```json
{
  "companyName": "Acme Corp",
  "contactEmail": "privacy@acme.com",
  "website": "https://acme.com",
  "jurisdiction": "EU",
  "outputDir": "legal",
  "outputFormat": "html",
  "language": "en",
  "jurisdictions": ["EU", "US-CA", "US-VA"],
  "dpoName": "Jane Smith",
  "dpoEmail": "dpo@acme.com",
  "euRepresentative": "EU Rep GmbH",
  "companyLocation": "Berlin, Germany",
  "dataRetentionDays": 365,
  "tollFreeNumber": "1-800-ACME",
  "securityEmail": "security@acme.com",
  "bugBountyUrl": "https://acme.com/security",
  "aiRiskLevel": "limited",
  "aiUsageDescription": "We use AI for content recommendations",
  "excludeServices": ["sentry"],
  "plugins": ["codepliant-plugin-healthcare"],
  "sectionOverrides": {
    "## Data Retention": "We retain data for 90 days unless otherwise required by law."
  }
}
```

### All Configuration Fields

| Field | Type | Description |
|-------|------|-------------|
| `companyName` | `string` | Company or app name (appears in all documents) |
| `contactEmail` | `string` | Contact email for privacy inquiries |
| `website` | `string` | Your website URL |
| `jurisdiction` | `string` | Primary legal jurisdiction |
| `outputDir` | `string` | Output directory (default: `"legal"`) |
| `outputFormat` | `string` | Default format: `markdown`, `html`, `pdf`, `json`, `all` |
| `language` | `string` | Document language: `en`, `es`, `fr`, `de` |
| `excludeServices` | `string[]` | Service names to exclude from scanning |
| `dpoName` | `string` | Data Protection Officer name (GDPR) |
| `dpoEmail` | `string` | DPO contact email |
| `euRepresentative` | `string` | EU representative name (non-EU companies) |
| `dataRetentionDays` | `number` | Data retention period in days |
| `aiRiskLevel` | `string` | AI risk classification: `minimal`, `limited`, `high` |
| `aiUsageDescription` | `string` | Custom description of AI usage |
| `jurisdictions` | `string[]` | Applicable jurisdictions (e.g., `["EU", "US-CA"]`) |
| `companyLocation` | `string` | Physical company location |
| `tollFreeNumber` | `string` | Toll-free phone number (CCPA requirement) |
| `securityEmail` | `string` | Security contact email (for SECURITY.md) |
| `bugBountyUrl` | `string` | Bug bounty program URL (for SECURITY.md) |
| `plugins` | `string[]` | Plugin package names to load |
| `sectionOverrides` | `object` | Override specific document sections by heading |

## MCP Server

Codepliant works as an [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) server, so AI coding tools like Claude Code and Cursor can scan projects and generate compliance docs directly.

### Setup

Add to your MCP config (`.mcp.json`, Claude Code settings, or Cursor settings):

```json
{
  "mcpServers": {
    "codepliant": {
      "command": "npx",
      "args": ["codepliant-mcp"]
    }
  }
}
```

### Available MCP Tools

| Tool | Description |
|------|-------------|
| `scan_project` | Scan a project and return detected services, data categories, and compliance needs |
| `generate_compliance_docs` | Scan and generate all compliance documents |
| `check_compliance` | Quick check if required compliance documents exist |

## GitHub Action

Add compliance checks to your CI/CD pipeline:

```yaml
# .github/workflows/compliance.yml
name: Compliance Check
on: [push, pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: codepliant/codepliant@v14
        with:
          fail-on-missing: true
```

### Inputs

| Input | Description | Default |
|-------|-------------|---------|
| `path` | Path to the project to scan | `.` |
| `output-dir` | Directory for compliance documents | `legal` |
| `fail-on-missing` | Fail if required docs are missing | `false` |
| `comment-on-pr` | Post scan results as a PR comment | `false` |

### Outputs

| Output | Description |
|--------|-------------|
| `services-found` | Number of third-party services detected |
| `documents-needed` | Number of compliance documents needed |
| `compliance-score` | Compliance score (0-100) |

Automatically check compliance before every commit:

Works with **husky**, **lefthook**, or **raw git hooks**. Prevents commits when required compliance documents are missing or outdated.

## EU AI Act

The EU AI Act (full enforcement August 2026) requires transparency about AI usage in applications. When Codepliant detects AI services in your codebase, it automatically generates:

**AI Disclosure** (`AI_DISCLOSURE.md`)
- AI systems in use and their providers
- What data is sent to AI services
- How AI-generated content is identified
- Third-party AI provider policies

**AI Act Compliance Checklist** (`AI_ACT_CHECKLIST.md`)
- Risk classification guidance (minimal, limited, high)
- Human oversight requirements
- Data processing transparency obligations
- Actionable checklist items

Set `"aiRiskLevel"` and `"aiUsageDescription"` in your config to customize the output.

## Programmatic API

Use Codepliant as a library:

```typescript
import { scan, generateDocuments, writeDocuments, loadConfig } from "codepliant";

const config = loadConfig("./my-project");
const result = scan("./my-project");
const docs = generateDocuments(result, config);
writeDocuments(docs, "./legal");
```

Available exports: `scan`, `generateDocuments`, `writeDocuments`, `loadConfig`, `saveConfig`, `writeDocumentsInFormat`, `writeMarkdown`, `writeHtml`, `getOutputFormat`, `generateBadge`, `writeBadges`, `installHook`, `uninstallHook`, `loadPlugins`.

Sub-path imports are also available:

```typescript
import { scan } from "codepliant/scanner";
import { generateDocuments } from "codepliant/generator";
import { writeDocumentsInFormat } from "codepliant/output";
import { loadConfig } from "codepliant/config";
```

```typescript
    {
      name: "HIPAA BAA",

    },
  ],

export default plugin;
```

## How It Works

```
Your Code --> Codepliant Scanner --> Document Generator --> /legal/
```

Three scan layers run across all ecosystems:

1. **Dependencies** ŌĆö reads manifest files (`package.json`, `requirements.txt`, `go.mod`, `Gemfile`, `Cargo.toml`, `composer.json`, `pom.xml`, `.csproj`) for known service packages
2. **Imports** ŌĆö scans source files for import statements, API usage patterns, and SDK initialization
3. **Environment** ŌĆö checks `.env` / `.env.example` for service API keys and configuration variables

Plus specialized scanners for Prisma schemas, Django models, SQLAlchemy models, Go structs, API routes, tracking pixels, framework-implicit services, industry compliance (HIPAA/PCI/COPPA), accessibility, and data flow analysis.

Everything runs locally. No data leaves your machine. No account required.

## Zero Network Calls

Codepliant makes **zero network calls**. None. Ever.

- **Everything runs locally.** Scanning, analysis, and document generation all happen on your machine using only the local filesystem.
- **No data leaves your machine.** Your source code, dependency manifests, environment variables, and generated documents never touch a remote server.
- **No telemetry, no analytics, no phone-home.** There is no usage tracking, crash reporting, or update checking. Codepliant does not know you are using it.
- **No account required.** There is no sign-up, no API key, no license server.
- **Verifiable.** The source code is open. Run the included `no-network.test.ts` test to programmatically verify that no network imports, fetch calls, or telemetry code exist in the codebase. The test scans every source file and will fail if any network-related code is introduced.

The only runtime dependency is the [MCP SDK](https://www.npmjs.com/package/@modelcontextprotocol/sdk), which communicates exclusively over **stdio** (standard input/output) ŌĆö not HTTP.

## Limitations

- Generated documents are starting points and **should be reviewed by a legal professional**
- Detection relies on known package names and patterns; custom or uncommon services may not be detected
- PDF output uses a text-based fallback when native PDF libraries are unavailable
